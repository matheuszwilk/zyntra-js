ZyntraJS integrates seamlessly with Next.js to provide end-to-end type safety between your React components and API layer. This guide walks you through building a complete full-stack application with the Next.js App Router.

The combination delivers the best of both worlds: Next.js's excellent developer experience with React Server Components, paired with ZyntraJS's automatic type inference and runtime safety. You'll write less boilerplate, catch more bugs at compile time, and ship features faster.

<Callout type="info" title="App Router Required">
  This guide uses Next.js 15+ with the App Router. The Pages Router is not supported.
</Callout>

---

## Manual Setup

<Card title="Complete Next.js Setup Guide">
  <Steps>
    <Step>
      ### Prerequisites

      Before you begin, ensure you have:
      - **Node.js** 22.0+
      - **Next.js** 15.0+ with App Router
      - **TypeScript** 5.0+
    </Step>

    <Step>
      ### Install Core Package

      Install ZyntraJS and Zod for schema validation:

<Tabs items={['npm', 'pnpm', 'yarn', 'bun']} groupId="package-manager">
  <Tab value="npm">
    ```bash
    npm install @zyntra-js/core zod
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    pnpm add @zyntra-js/core zod
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn add @zyntra-js/core zod
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun add @zyntra-js/core zod
    ```
      </Tab>
    </Tabs>

      Zod provides excellent TypeScript inference and runtime validation, making it the recommended choice for ZyntraJS projects.
    </Step>

    <Step>
      ### Define Application Context

The context defines dependencies available throughout your API (database, services, etc.). Create `src/zyntra.context.ts`:

```typescript title="src/zyntra.context.ts"
/**
 * Application Context Type
 * Defines what dependencies are available to all API handlers
 */
export interface AppContext {
  // Add your dependencies here, e.g.:
  // db: PrismaClient
  // auth: AuthService
}

/**
 * Context Factory
 * Creates the context instance passed to handlers
 */
export function createZyntraAppContext(): AppContext {
  return {
    // Initialize dependencies here
  }
}
      ```
    </Step>

    <Step>
      ### Initialize ZyntraJS

Create the main Zyntra instance with configuration. Create `src/zyntra.ts`:

```typescript title="src/zyntra.ts"
import { Zyntra } from '@zyntra-js/core'
import { createZyntraAppContext } from './zyntra.context'

export const zyntra = Zyntra
  .context(createZyntraAppContext())
  .config({
    baseURL: process.env.NEXT_PUBLIC_ZYNTRA_API_URL || 'http://localhost:3000',
    basePATH: process.env.NEXT_PUBLIC_ZYNTRA_API_BASE_PATH || '/api/v1',
  })
  .docs({
    info: {
      title: 'My Next.js API',
      version: '1.0.0',
      description: 'API built with ZyntraJS',
    }
  })
  .create()
      ```

      The builder pattern makes it easy to add features later like `.store()` (Redis), `.jobs()` (background tasks), `.logger()`, and `.telemetry()`.
    </Step>

    <Step>
      ### Create Your First Controller

Controllers group related endpoints. Create `src/features/user/controllers/user.controller.ts`:

```typescript title="src/features/user/controllers/user.controller.ts"
import { zyntra } from '@/zyntra'

export const userController = zyntra.controller({
  name: 'User',
  path: '/users',
  actions: {
    list: zyntra.query({
      path: '/',
      handler: async ({ response }) => {
        return response.success({
          users: [
            { id: 1, name: 'Alice', email: 'alice@example.com' },
            { id: 2, name: 'Bob', email: 'bob@example.com' },
          ]
        })
      },
    }),
  },
})
```

Create barrel export at `src/features/user/index.ts`:

```typescript title="src/features/user/index.ts"
      export * from './controllers/user.controller'
      ```

      This creates `GET /api/v1/users/` with fully typed responses.
    </Step>

    <Step>
      ### Create Application Router

The router assembles all controllers. Create `src/zyntra.router.ts`:

```typescript title="src/zyntra.router.ts"
import { zyntra } from './zyntra'
import { userController } from './features/user'

export const AppRouter = zyntra.router({
  controllers: {
    user: userController
  }
})

      export type AppRouterType = typeof AppRouter
      ```

      The type export enables the type-safe client without exposing server implementation.
    </Step>

    <Step>
      ### Create Next.js API Route

Connect the router to Next.js using the adapter. Create `src/app/api/v1/[[...all]]/route.ts`:

```typescript title="src/app/api/v1/[[...all]]/route.ts"
import { AppRouter } from '@/zyntra.router'
import { nextRouteHandlerAdapter } from '@zyntra-js/core/adapters'

export const { GET, POST, PUT, DELETE } = nextRouteHandlerAdapter(AppRouter)
```

The `[[...all]]` catch-all route handles all methods and paths under `/api/v1/`.

### Step 7: Generate Type-Safe Client

Create the client for frontend usage at `src/zyntra.client.ts`:

```typescript title="src/zyntra.client.ts"
import { createZyntraClient } from '@zyntra-js/core/client'
import type { AppRouterType } from './zyntra.router'

export const api = createZyntraClient<AppRouterType>({
  baseURL: process.env.NEXT_PUBLIC_ZYNTRA_API_URL || 'http://localhost:3000',
  basePATH: process.env.NEXT_PUBLIC_ZYNTRA_API_BASE_PATH || '/api/v1',
  router: () => {
    if (typeof window === 'undefined') {
      return require('./zyntra.router').AppRouter
    }
    return require('./zyntra.schema').AppRouterSchema
  },
})

export type ApiClient = typeof api
```

      This client works in both Server and Client Components with full type safety.
    </Step>
  </Steps>
</Card>

## Usage in Components

### Server Components

Use direct API calls in Server Components:

```typescript title="app/page.tsx"
import { api } from '@/zyntra.client'

export default async function HomePage() {
  const { users } = await api.user.list.query()

  return <div>Users: {users.length}</div>
}
```

### Client Components

Use React hooks for automatic loading/error states:

```typescript title="components/user-list.tsx"
'use client'

import { api } from '@/zyntra.client'

export function UserListComponent() {
  const { data, isLoading, error } = api.user.list.useQuery()

  if (isLoading) return <div>Loading...</div>
  if (error) return <div>Error: {error.message}</div>

  return <div>Users: {data.users.length}</div>
}
```

## Generating Schema and Docs

ZyntraJS can automatically generate OpenAPI schemas and TypeScript types for your API.

### When to Generate Schema

<Callout type="error">
  **Critical:** You MUST run schema generation BEFORE using the client in your frontend. The client depends on the generated schema to function correctly.
</Callout>

<Callout type="info">
  Run this command whenever you modify your API structure (add/remove/modify controllers or actions) OR before using the client for the first time.
</Callout>

### Generate Schema

Run this command whenever you make changes to your API structure:

<Tabs items={['npm', 'pnpm', 'yarn', 'bun']} groupId="generate-schema">
  <Tab value="npm">
    ```bash
    npx @zyntra-js/cli@latest generate schema
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    pnpm dlx @zyntra-js/cli@latest generate schema
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn dlx @zyntra-js/cli@latest generate schema
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bunx @zyntra-js/cli@latest generate schema
    ```
  </Tab>
</Tabs>

This creates:
- `src/zyntra.schema.ts` - Type-safe schema for client-side usage
- `src/docs/openapi.json` - OpenAPI 3.0 specification

### Update Zyntra Config

Import the generated OpenAPI schema in `src/zyntra.ts`:

```typescript title="src/zyntra.ts"
import { Zyntra } from '@zyntra-js/core'
import { createZyntraAppContext } from './zyntra.context'
import openapi from './docs/openapi.json'

export const zyntra = Zyntra
  .context(createZyntraAppContext())
  .config({
    baseURL: process.env.NEXT_PUBLIC_ZYNTRA_API_URL || 'http://localhost:3000',
    basePATH: process.env.NEXT_PUBLIC_ZYNTRA_API_BASE_PATH || '/api/v1',
  })
  .docs({
    openapi, // ‚Üê Add generated OpenAPI
    info: {
      title: 'My Next.js API',
      version: '1.0.0',
      description: 'API built with ZyntraJS',
    }
  })
  .create()
```

## Zyntra Studio (API Playground)

ZyntraJS includes an interactive API playground called **Zyntra Studio**, powered by Scalar. It's automatically available at `/api/v1/docs` when you configure the `.docs()` method.

To access it:

1. Start your dev server: `npm run dev`
2. Navigate to `http://localhost:3000/api/v1/docs`
3. You'll see an interactive interface where you can:
   - Browse all API endpoints
   - View request/response schemas
   - Test endpoints directly in the browser
   - See auto-generated examples

The playground updates automatically as you add new endpoints or modify existing ones. It's perfect for API development and testing without needing external tools like Postman.
