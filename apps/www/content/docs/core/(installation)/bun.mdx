Bun is a modern JavaScript runtime that's incredibly fast, with built-in bundling, testing, and package management. When combined with ZyntraJS, you get exceptional performance with zero configuration窶馬o build tools, no bundlers, just pure speed and developer happiness.

This guide shows you how to build production-ready APIs with Bun and ZyntraJS. You'll experience Bun's lightning-fast startup times, instant hot reload, and native TypeScript support, all while maintaining ZyntraJS's type-safe controllers and automatic API documentation.

<Callout type="success" title="Maximum Performance">
  Bun + ZyntraJS delivers incredible performance out of the box. Bun's runtime is 4x faster than Node.js, with instant startup and built-in TypeScript support.
</Callout>

## Manual Setup

<Card title="Complete Bun Setup Guide">
  <Steps>
    <Step>
      ### Prerequisites

      Before you begin, ensure you have:
      - **Bun** 1.0+ ([install here](https://bun.sh))
      - Basic knowledge of TypeScript
    </Step>

    <Step>
      ### Install Dependencies

      Install ZyntraJS and Zod for schema validation:

      ```bash
      bun add @zyntra-js/core zod
      ```

      Bun has native TypeScript support窶馬o additional type packages or build tools needed!
    </Step>

    <Step>
      ### Define Application Context

The context defines dependencies available throughout your API. Create `src/zyntra.context.ts`:

```typescript title="src/zyntra.context.ts"
/**
 * Application Context Type
 * Defines what dependencies are available to all API handlers
 */
export interface AppContext {
  // Add your dependencies here, e.g.:
  // db: Database
  // cache: RedisClient
}

/**
 * Context Factory
 * Creates the context instance passed to handlers
 */
export function createZyntraAppContext(): AppContext {
  return {
    // Initialize dependencies here
  }
}
      ```
    </Step>

    <Step>
      ### Initialize ZyntraJS

      Create the main Zyntra instance. Create `src/zyntra.ts`:

```typescript title="src/zyntra.ts"
import { Zyntra } from '@zyntra-js/core'
import { createZyntraAppContext } from './zyntra.context'

export const zyntra = Zyntra
  .context(createZyntraAppContext())
  .config({
    baseURL: process.env.ZYNTRA_API_URL || 'http://localhost:3000',
    basePATH: process.env.ZYNTRA_API_BASE_PATH || '/api/v1',
  })
  .docs({
    info: {
      title: 'My Bun API',
      version: '1.0.0',
      description: 'High-performance API built with Bun and ZyntraJS',
    }
  })
  .create()
      ```
    </Step>

    <Step>
      ### Create Your First Controller

Controllers group related endpoints. Create `src/features/user/controllers/user.controller.ts`:

```typescript title="src/features/user/controllers/user.controller.ts"
import { zyntra } from '@/zyntra'

export const userController = zyntra.controller({
  name: 'User',
  path: '/users',
  actions: {
    list: zyntra.query({
      path: '/',
      handler: async ({ response }) => {
        return response.success({
          users: [
            { id: 1, name: 'Alice', email: 'alice@example.com' },
            { id: 2, name: 'Bob', email: 'bob@example.com' },
          ]
        })
      },
    }),
  },
})
```

Create barrel export at `src/features/user/index.ts`:

```typescript title="src/features/user/index.ts"
export * from './controllers/user.controller'
```

      This creates `GET /api/v1/users/` with fully typed responses.
    </Step>

    <Step>
      ### Create Application Router

The router assembles all controllers. Create `src/zyntra.router.ts`:

```typescript title="src/zyntra.router.ts"
import { zyntra } from './zyntra'
import { userController } from './features/user'

export const AppRouter = zyntra.router({
  controllers: {
    user: userController
  }
})

      export type AppRouterType = typeof AppRouter
      ```
    </Step>

    <Step>
      ### Set Up Bun Server

Bun has a built-in HTTP server that's incredibly fast. Create `src/index.ts`:

```typescript title="src/index.ts"
import { serve } from 'bun'
import { AppRouter } from './zyntra.router'

const ZYNTRA_API_BASE_PATH = process.env.ZYNTRA_API_BASE_PATH || '/api/v1/'

const server = serve({
  routes: {
    // Mount ZyntraJS router
    [ZYNTRA_API_BASE_PATH + '*']: AppRouter.handler,
  },
})

console.log(`泅 Server running at ${server.url}`)
```

Bun's native routing is simple and performant窶馬o framework overhead needed!

### Step 7: Add Development Scripts

Update your `package.json`:

```json title="package.json"
{
  "scripts": {
    "dev": "bun --hot src/index.ts",
    "start": "bun src/index.ts"
  }
}
```

      Now run `bun dev` for instant hot reload! Changes apply in milliseconds.
    </Step>
  </Steps>
</Card>

## Bun-Specific Features

### Native TypeScript

Bun runs TypeScript directly窶馬o transpilation needed:

```typescript
// Just write TypeScript and run it!
import type { User } from './types'

const user: User = { name: 'John' }
```

### Environment Variables

Bun has built-in `.env` support. Create `.env`:

```bash title=".env"
ZYNTRA_API_BASE_PATH=/api/v1/
ZYNTRA_API_URL=http://localhost:3000
```

Access them via `Bun.env`:

```typescript
const port = Bun.env.PORT || 3000
```

### Built-in Testing

Bun includes a fast test runner compatible with Jest:

```typescript title="src/features/user/user.test.ts"
import { describe, test, expect } from 'bun:test'
import { api } from '@/zyntra.client'

describe('User Controller', () => {
  test('returns list of users', async () => {
    const result = await api.user.list.query()
    expect(result.users.length).toBeGreaterThan(0)
  })
})
```

Run tests with `bun test` - it's incredibly fast!

## Full-Stack with Bun React

For full-stack applications, use the `bun-react-app` template which includes React:

```typescript title="src/index.tsx" {10}
import { serve } from 'bun'
import { AppRouter } from './zyntra.router'
import index from './index.html'

const server = serve({
  routes: {
    // Serve React app for all unmatched routes
    '/*': index,
    
    // Serve API
    '/api/v1/*': AppRouter.handler,
  },
  
  development: Bun.env.NODE_ENV !== 'production' && {
    hmr: true,       // Hot module reload
    console: true,   // Browser console logging
  },
})

console.log(`泅 Server running at ${server.url}`)
```

This single file serves both your React frontend and ZyntraJS API with hot reload!

## Type-Safe Client

Create a type-safe client for consuming your API. Create `src/zyntra.client.ts`:

```typescript title="src/zyntra.client.ts"
import { createZyntraClient } from '@zyntra-js/core/client'
import type { AppRouterType } from './zyntra.router'

export const api = createZyntraClient<AppRouterType>({
  baseURL: Bun.env.ZYNTRA_API_URL || 'http://localhost:3000',
  basePATH: Bun.env.ZYNTRA_API_BASE_PATH || '/api/v1',
  router: () => require('./zyntra.router').AppRouter,
})

export type ApiClient = typeof api
```

Use it anywhere:

```typescript
// Fully typed, instant autocomplete
const { users } = await api.user.list.query()
```

## Generating Schema and Docs

For full-stack Bun projects (Bun + React), you'll need to generate the client schema whenever you modify your API.

### When to Generate Schema

<Callout type="error">
  **Critical:** You MUST run schema generation BEFORE using the client in your frontend. The client depends on the generated schema to function correctly.
</Callout>

<Callout type="info">
  Run this command whenever you modify your API structure (add/remove/modify controllers or actions) OR before using the client for the first time.
</Callout>

### Generate Schema

Run this command whenever you make changes to your API structure:

## Zyntra Studio (API Playground)

ZyntraJS includes an interactive API playground called **Zyntra Studio**. It's automatically available at `/api/v1/docs` when you configure the `.docs()` method.

To access it:

1. Start your dev server: `bun dev`
2. Navigate to `http://localhost:3000/api/v1/docs`
3. You'll see an interactive interface where you can:
   - Browse all API endpoints
   - View request/response schemas
   - Test endpoints directly
   - See auto-generated examples

The playground updates automatically as you add endpoints.

## Performance Tips

### Use Bun's Native APIs

Bun provides optimized native APIs for common tasks:

```typescript
// File I/O (much faster than Node.js fs)
const file = Bun.file('data.json')
const data = await file.json()

// Hashing
const hash = Bun.hash('sha256', 'data')

// Environment variables
const secret = Bun.env.SECRET_KEY
```

### Database with Bun

Bun works great with Prisma, the recommended ORM for ZyntraJS:

```typescript
import { PrismaClient } from '@prisma/client'
const prisma = new PrismaClient()
```
