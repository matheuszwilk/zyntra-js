## Overview

This template provides a robust foundation for building full-stack, type-safe applications with Next.js and ZyntraJS. It leverages the latest Next.js App Router conventions and integrates essential services for modern web development.

## Features

- ✅ **Next.js App Router**: Built with the latest Next.js conventions for server and client components.
- ✅ **End-to-End Type Safety**: Powered by ZyntraJS, ensuring type safety from your React components to your back-end API.
- ✅ **Feature-Based Architecture**: A scalable project structure that organizes code by business domain.
- ✅ **Ready-to-Use Services**: Pre-configured examples for:
  - **Caching**: Integrated with Redis via `@zyntra-js/adapter-redis`.
  - **Background Jobs**: Asynchronous task processing with BullMQ via `@zyntra-js/adapter-bullmq`.
  - **Structured Logging**: Production-ready logging.
- ✅ **Database Ready**: Comes with Prisma set up for seamless PostgreSQL database integration.
- ✅ **Seamless Integration**: Uses the `nextRouteHandlerAdapter` to cleanly connect the ZyntraJS router to the Next.js App Router.
- ✅ **UI Components**: Includes a set of UI components from `shadcn/ui` to get you started quickly.

## Key Features Explained

### Next.js API Route Handler

The API entry point is `src/app/api/[[...all]]/route.ts`, which uses `nextRouteHandlerAdapter` from ZyntraJS to expose the API.

```typescript
// src/app/api/[[...all]]/route.ts
import { AppRouter } from '@/zyntra.router'
import { nextRouteHandlerAdapter } from '@zyntra-js/core/adapters'

export const { GET, POST, PUT, DELETE } = nextRouteHandlerAdapter(AppRouter)
```

### ZyntraJS API Layer

Backend logic is defined using ZyntraJS:
- **Initialization (`src/zyntra.ts`)**: Core Zyntra instance configured with adapters for store (Redis), jobs (BullMQ), logging, and telemetry.
- **Router (`src/zyntra.router.ts`)**: Defines all API controllers.
- **Controllers (`src/features/[feature]/controllers/`)**: Group related API actions (queries and mutations).

### Type-Safe Client & React Hooks

ZyntraJS automatically generates a type-safe client (`src/zyntra.client.ts`) for seamless interaction with your backend from both Server and Client Components.

**Server Component Usage:**
```tsx
// app/some-server-page/page.tsx
import { api } from '@/zyntra.client';

export default async function SomePage() {
  const data = await api.example.health.query();
  return <div>Status: {data.status}</div>;
}
```

**Client Component Usage:**
```tsx
// components/SomeClientComponent.tsx
'use client';
import { api } from '@/zyntra.client';

export function SomeClientComponent() {
  const { data, isLoading } = api.example.health.useQuery();

  if (isLoading) return <div>Loading...</div>;
  return <div>Status: {data?.status}</div>;
}
```
